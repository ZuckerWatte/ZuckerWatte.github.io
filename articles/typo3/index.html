Typo3

Written by Mona

Ein kurzer Überblick über das Content-Management-System Typo3.

Gründe für Frameworks: Sicherheitsaspekte, sauberer Request/Response Flow (MVC-Pattern), Wiederverwendbarkeit, Organisation, Bundles (Libraries), –> Produktivität

“Build your App, not your Tools”

Symfony Request Flow: Request an Front Controller geht an Symfony Kernel = Routing, Rendering, Events auslösen (z.B. Mails versenden), Firewall sendet Response

Request -> ChainRouter(Nutzerverwaltung) oder DynamicRouter(CMS Content) -> Controller -> Model -> View -> Response

Nachteile von Frameworks: ohne Caching im System sehr langsam (bootstrap on every request), komplexe Codebase –> steile Lernkurve

Symfony Overview:

  • request/response flow
  • full stack or single components
  • common patterns

Full Stack: Composed of components (HTTPFoundation, Routing, Form, Twig Templating, Form, ..)

PHPCR

  • Implementation of JCR
  • kompatibel zu Java
  • standardisierte API
  • Features: tree access (hierarchischer content, jeder node object in DB), access by uuid, versioning, locking, xmp import and export (einfaches Umziehen von Daten, ohne lästige Migrationen), search nodes
  • “RDBMS are not a good fit for CMS Data, that is mostly unstructured, hurray for PHPCR”

Symfony CMF

  • set of Symfony bundles (routing, menu, ..)
  • Base for CMS functionality
  • leichtes nachbauen von CMS-Funktionalität in bestehenden Applicationen
  • Code sharing, well tested

Symfony in CMS:

SULU

  • symfony full stack
  • semantische Konfiguration von Templates in XML
  • live preview
  • eigene Modul mit bundles dazu bauen
  • http://sulu.io

EZ Platform

  • full stack symfony
  • uses symfony CMF Router
  • flexible content elements
  • Documents saved as DocBook (XML-Format) –> leider nicht PHPCR Standard, sondern eigen
  • strukturiert in bundles
  • ezplatform.com

Drupal 8

  • symfony componenets
  • Altlasten bleiben bestehen
  • Rich Admin interface editors (Redakteure mehr Freiheit)
  • array of doom theme config

NEOS “Typo3 Community hat Probleme”

  • eigenes Templating
  • eigenes Framework
  • eigene PHPCR implementation
  • sollte Typo3 Version 5 werden

TYPO3

  • Typo3 Version 7
  • große Community
  • “technologisch auf schlechtem Fuß”
  • alte und komplexe Codebase
  • schlechtes Datenbank design
  • symfony console crippled (Eigenentwicklung)
